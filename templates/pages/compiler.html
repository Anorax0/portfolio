About Project

Excel Compiler

At the beginning

One of processes in my work required from person doing it to download some data files, restructurize them, search for certain values, cross-check etc. This part sometimes took up to 4-5 hours. And then everything was checking in validation system in other tool as a next step in this process.

If everything was done correctly at first time, entire task for this process took up to 6 hours. But there was a problem. Everytime.

Normally this took up to 12 hours. Not great, not terrible. Nope, very terrible. Nightmare.

I decided to solve it somehow.

How it works

It is semi-automated process, due to closed API of other tools. However, script shorten time of entire task from about 12 hours to just 1 hour, where running app tooks up to 1.5 second to prepare required documents.

This step starts with data preparation. User has to download needed files manually and save it in certain directory. Let's say, we can sum records from all the inputed files to 8 thousands. And run the app.

Secondly, user has to type with which asset he/she would like to work, and then type 1 of 2 mode of data, he/she would like to receive.

After 1, up to 1.5 seconds, user has the output. The app in saving data in known structures, rearrange them, searches for values, validate and cross-check. If some record is incorrect, user will know about it. Outputed file contains inputed data, some of the critical single steps and almost ready-to-use data.

Thirdly, outputed data has to be checked due to some incorrect data - normally it is limited to visual check of records.

Finally, user can user app to extract correct data from above files to receive ready-to-use spreadsheet required in external tool.

Application in written in Python with Openpyxla library. Runs in console, but exported with PyInstaller to exe file. Using a Colorama and Termcolor for output. Due to internal restrictions I cannot show any piece of application.
